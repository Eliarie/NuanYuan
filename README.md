# PRD：暖园 · 家园共育系统

## Monorepo 结构

- `packages/client`：前端小程序 / H5 入口
- `packages/shared`：前后端共享类型、DTO 和 Zod Schema
- `packages/server`：后端 API 与业务服务入口

当前仓库已经按上述结构整理，前端业务代码集中放在 `packages/client`，共享定义集中放在 `packages/shared`。

---

## 一、核心价值主张

> 让教师"说说话"就能完成家园沟通，让家长"看得懂"孩子在幼儿园的真实状态。

传统家园沟通的矛盾：教师每天要写大量文字反馈，精力耗尽；家长收到的是模板化内容，感受不到孩子的真实成长。

暖园的解法：**AI把教师的"随口一说"变成有价值的家园沟通内容**，同时双向收集信息，共同构建每个孩子的动态成长画像。

---

## 二、前端项目启动指南（本地编译与运行）

本项目采用标准的 Monorepo 架构 (基于 `npm workspaces`)，包含客户端代码与共享类型定义。

### 1. 环境准备

1. **Node.js**: 推荐版本 `>= 18.0.0`
2. **包管理器**: 使用传统的 `npm` (Node.js 自带)
3. **开发工具**: 推荐使用 VS Code (配套 Taro 插件) 以及 微信开发者工具。

### 2. 安装依赖

在项目根目录下，运行以下命令安装所有工作区依赖：

```bash
npm install
```

### 3. 启动开发服务器

前端项目位于 `packages/client` 目录下。你可以通过 npm workspaces 的 `-w` 参数来运行前端的启动脚本。

#### 本地开发 (H5)

推荐优先使用 H5 模式进行快速 UI 调试和交互验证。

```bash
# 在根目录运行 client 的 h5 开发模式
npm run dev:h5 -w client
```

或者进入前端目录运行：

```bash
cd packages/client
npm run dev:h5
```

运行后，通常可以在浏览器中访问 `http://127.0.0.1:10086/` 预览应用。

#### 小程序开发与预览

若需验证原生小程序 API 或准备提审，可运行微信小程序编译：

```bash
# 生成供微信开发者工具使用的小程序代码
npm run dev:weapp -w client
```

或者进入前端目录运行：

```bash
cd packages/client
npm run dev:weapp
```

编译完成后，打开 **微信开发者工具**，导入位于 `packages/client/dist` 目录下的产物。
注意：请在微信开发者工具中填写你真实的 AppID 或使用测试号。

### 4. 生产环境构建

构建生产环境产物：

```bash
# 构建 H5 生产产物
npm run build:h5 -w client

# 构建 小程序 生产产物
npm run build:weapp -w client
```

## 三、用户角色与核心痛点

| 角色 | 场景 | 核心痛点 |
|------|------|----------|
| 幼儿园教师 | 一天带20+孩子，下班前还要写家园反馈 | 工作量大、情绪耗竭、不知道怎么跟家长说"问题孩子" |
| 幼儿家长 | 每天接孩子只能问"今天怎么样" | 信息不透明、不知道孩子在园真实状态、不知道如何配合老师 |
| 幼儿（核心对象） | 在园和在家是两个世界 | 成长信息割裂，没有人能看到完整的他/她 |

## 四、核心功能模块

### 模块 1：教师 AI 树洞（核心交互）

**触发场景**：

- 遇到某个孩子的行为问题，不知道如何处理
- 一天结束，情绪耗竭，需要倾诉

**AI 对话三层响应**：

1. **情绪层**：先共情、安慰，让教师感到被理解
2. **思考层**：引导教师更深层分析行为背后的原因（*"你觉得他为什么会这样？家里最近有什么变化吗？"*）
3. **策略层**：给出具体可操作的班级管理建议或应对方法

**跨会话记忆机制**：

- AI 记住每个孩子的历史记录，下次提到同一个孩子时主动衔接：*"上次你说小明打人，你当时打算试试冷处理，后来怎么样了？"*
- 教师不需要回忆和重复背景，AI 帮他串联上下文
- 如果策略有效，AI 记录并强化；如果无效，AI 引导调整方向

**后台静默完成**：

- 提取对话中涉及的孩子信息 → 更新成长画像和个案时间线
- 判断是否需要家长配合 → 起草推送内容供教师确认

**专业成长副产品**：对于专业度不足的教师，AI 在给策略的同时潜移默化传递幼教专业知识，帮助教师提升能力

### 模块 2：幼儿成长画像 + 个案观察

**基础档案**：

- 每个孩子一张动态档案卡
- 维度：社交发展 / 情绪管理 / 认知能力 / 生活习惯 / 特别事件
- 数据来源：多名教师输入（在园，综合不同视角）+ 多名家长输入（在家，如父母长辈）双向构建
- 教师可查看全班，家长只可查看自己孩子

**个案观察时间线**（自动生成，无需教师额外操作）：

- 系统从树洞对话中自动提取，按孩子维度整理成时间线（综合多名教师的观察记录）
- 每条记录包含：问题描述 / 分析原因 / 选择的策略 / 后续跟进结果
- 示例：

  ```
  小明 · 个案记录
  09-15  问题：打人行为，情绪失控
         策略：冷处理 + 事后复盘
  09-28  跟进：冷处理有效，但需要一致性
         新问题：开始抢玩具
         策略：角色扮演练习分享
  10-12  进展：抢玩具减少，情绪稳定
         家长反馈：在家也有改善
  ```

**学期末个案报告**（一键生成）：

- 整合该学期所有相关对话和记录
- 输出完整文本档案：问题历程 / 尝试过的策略 / 有效与无效的方法 / 家长配合情况 / 成长结论
- 可导出，作为幼儿成长档案留存或交接给下一任教师

### 模块 3：家长端

**触发场景**：

- 收到推送通知，想了解孩子今天发生了什么
- 主动打开，关心孩子近期在园状态
- 遇到育儿困惑，不知道怎么处理孩子的某个行为
- 想告诉老师孩子在家的情况

**两种进入方式**：

- **被动接收**：教师确认后，系统推送"需要家长配合"的内容，说明孩子情况 + 具体配合建议（*"小明最近在园有些情绪波动，建议在家可以这样做……"*）
- **主动查看**：随时打开查看孩子近期成长动态，也可以向 AI 咨询育儿问题

**AI 对话支持**：

1. **解读层**：帮家长理解孩子行为背后的原因（*"3-4岁孩子出现这种行为很常见，通常是因为……"*）
2. **建议层**：给出在家可以配合的具体做法
3. **倾听层**：家长也可以倾诉育儿压力，AI 提供情绪支持

**家长反馈**：

- 补充孩子在家的表现、可能的原因
- 对推送内容有疑问可直接提问
- 所有输入自动同步教师端 + 更新孩子画像

### 模块 4：家长回顾

**孩子成长维度**：

- 本月 / 本学期孩子在园的成长轨迹
- 教师记录的关键事件和进步节点
- 孩子在家 vs 在园的表现对比，发现规律

**家园协作维度**：

- 收到过哪些配合建议，自己做了什么
- 哪些方法有效，哪些需要调整
- 与教师的沟通记录回顾

---

### 模块 5：教师回顾

**班级维度**：

- 本月 / 本学期全班幼儿成长概览
- 哪些孩子有明显进步，哪些需要持续关注
- 班级整体发展趋势

**教师自身维度**：

- 本月 / 本学期处理了哪些典型问题
- AI 给过哪些建议，效果如何
- 专业能力成长轨迹（处理问题的思路是否在进步）

## 五、关键用户流程

```
教师流程：
打开小程序 → 对话框吐槽 → AI安慰 + 后台记录
→ AI起草家长推送（教师确认）→ 发送给家长

家长流程：
收到推送通知 → 查看孩子今日情况 → 可选：补充在家表现
→ 信息同步给教师 + 更新孩子画像

孩子画像构建：
教师输入 ＋ 家长输入 → AI整合 → 动态成长档案
（随时间积累，形成完整的孩子成长记录）
```

## 六、产品差异化亮点

1. **情绪优先设计**：入口不是"填表"，是"说话"——降低教师使用门槛，顺带完成信息采集
2. **AI做翻译层**：教师的原始表达（可能带情绪）→ AI转化为家长友好的沟通语言
3. **双向画像构建**：不是单向通知，是真正的信息闭环，孩子画像越来越完整
4. **减负而非增负**：教师不需要额外学习新工作流，"吐槽"本身工作

## 七、MVP 范围建议

**先做这一件事**：教师吐槽 → AI整理 → 推送给家长

砍掉：画像可视化、家长反馈闭环、数据统计（放 v2）

**验证核心假设**：教师愿意用"说话"代替"写反馈"吗？

## 八、技术平台

**终端**：微信小程序（教师端 + 家长端均在微信内）

微信小程序的优势：

- 无需下载安装，家长和教师开箱即用
- 微信推送通知天然触达，无需额外通知权限
- 微信账号体系直接复用，降低注册门槛

## 九、关键决策

| 问题 | 决策 |
|------|------|
| 终端 | 微信小程序（教师端 + 家长端） |
| 账号绑定 | 教师创建班级 + 录入孩子名单，生成每个孩子的专属邀请码，支持多位家长（如爸爸、妈妈、爷爷等）扫码/填码共同绑定同一个孩子 |
| 班级与教师管理 | 支持多班级架构，一个班级可绑定多名教师共管，AI 能够综合多名教师对同一个孩子的观察记录 |
| MVP 范围 | 单园版，先深度服务一所幼儿园，验证核心假设后再扩展 |
| 数据隐私 | 家长只可见自己孩子的画像；教师原始吐槽内容不对家长展示，仅展示 AI 整理后的版本 |
